Fundamental Rule: The algorithm MUST remain strictly Greedy (constructive only, NO backtracking, NO local search, NO metaheuristics like Tabu Search, NO external solvers). Even if the solution is not optimal, it must be fast and make reasonable local choices.
Task 1: Open IHTC_Greedy.cc. Create a helper function void assignNursesGreedy(const IHTC_Input& in, IHTC_Output& out);.
Task 2: Implement the function using a constructive approach: iterate over each day, each shift, each room. If the room is occupied on that day/shift (Hard constraint H8), find the best available nurse.
Task 3: The "Greedy" choice of the nurse must first filter for those who are on shift (checking their roster). Among the available ones, choose the one that minimizes the impact on Soft constraints (e.g., pick the nurse with the lowest current workload to minimize S4, or the one with skills closest to the requirement to minimize S2). Once chosen, save this assignment in the new nurse data structure inside IHTC_Output created previously.
Task 4: Modify the main function GreedyIHTCSolver to call assignNursesGreedy(in, out); immediately after scheduleInOrder.
